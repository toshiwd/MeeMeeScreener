# Purpose

`財務` タブを開いたときに、現在の細い右サイドバーではカード数値とグラフが潰れて読みにくい。今回の目的は、詳細画面の `財務` タブだけを専用レイアウトへ切り替え、画面の約半分まで使って EDINET 指標と推移グラフを一目で読めるようにすることだ。

変更後は、詳細画面で `財務` タブを押すと、左に価格チャート、右に広い財務パネルが並ぶ。財務パネルでは主要6指標が大きなカードで並び、売上・利益推移と利益率・ROE/ROA 推移が圧迫感なく読める。動作確認は、フロントをビルドして `財務` タブを開き、右ペインが通常メモ欄より明確に広くなること、カード数値が折り返し崩れしないこと、2つのグラフ凡例と軸が読めることを見る。

# Implementation Notes

`app/frontend/src/routes/DetailView.tsx` で `detail-content` に `with-memo-panel` と `detail-content-financial` を付け分ける。`showFinancialPanel` が真のときだけ、財務パネルの外側レイアウトを通常パネルから分離する。

`app/frontend/src/styles.css` では、後半にある `detail-content` の定義群が実際のレイアウトを支配している。ここにより強いセレクタで `detail-content.with-memo-panel.detail-content-financial` を追加し、右カラムを `minmax(560px, 46vw)` 程度で確保する。財務パネル自体は `width: auto` に戻し、カードは3列、グラフはより大きい角丸パネルとして表示する。

カードの視認性改善では、値の `overflow-wrap` を有効にし、負債のような長い数値が枠外へはみ出さないようにする。グラフは高さを右ペイン幅に合わせて 260px 以上にし、凡例文字を少し大きくする。メタ情報はヘッダ下のピルでまとめる。
