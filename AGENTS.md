# AGENTS.md (Global Guardrails)

目的:
- クレジット消費と暴走（無限探索・無限リトライ・無駄なテスト実行）を抑えつつ、最小変更で確実に前進する。

## 0) 最重要: コンテキスト節約
- 同時に開くタブ/ファイルは最大5つ。不要になったら即閉じる。
- 巨大ファイル（ログ/CSV/JSON/生成物/dist/.next/node_modules 等）は開かない。
  - 必要な行は `rg` で抽出し、該当行のみを提示する。

## 1) 進め方（強制）
- 必ず: Plan -> Evidence -> Execute -> Verify の順で進める。
- 1 fix = 1 symptom。無関係な整形/リネーム/大量置換を混ぜない。
- 依頼が曖昧な場合でも、まず repo を検索して「現状の証拠」を集め、勝手に仕様を変えない。

## 2) 実行（コマンド/テスト）ポリシー
- lint/test/build は「明示指示がある時だけ」実行する（勝手に回さない）。
- 同じ失敗コマンドを 2 回以上繰り返さない。
  - 2回失敗したら停止し、根因候補を1つに絞って「証拠 + 次の1手」を提示する。

## 3) Agent(Full Access) の暴走防止
- 無限に探索/修正/実行しない。3回連続で進捗がない場合は作業を止める。
- 新規にファイルを大量生成しない（必要なら最小単位）。

## 4) Cloud 委任
- デフォルト禁止。必要なら「理由」「範囲」「停止条件」を先に明記する。

## 5) ドキュメントの読み方
- 詳細仕様/Runbook は docs/CODEX.md にある（必要時だけ開く）。
- backend/front の領域ルールは各ディレクトリの AGENTS.md を優先する:
  - app/backend/AGENTS.md
  - app/frontend/AGENTS.md
