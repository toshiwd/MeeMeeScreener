# note記事4本を使った売買手法の再現性研究

## 概要

この研究は、`銘柄選定条件` の抽出ではなく、note記事に記録された `月足の前提 -> 日足シグナル -> 玉操作 -> 撤退条件` を局面単位で並べ替え、あとから自分の売買記録や過去チャートに当てて再現性を検証できる形にすることを目的とする。

今回の対象記事:

- [大平洋金属5541トレードレポート2025年9-12月](https://note.com/origindiamond/n/n985d5707bc8a)
- [システナ2317トレードレポート12月〜](https://note.com/origindiamond/n/n21f0432e9904)
- [宝HD2531トレードレポート12月〜12月](https://note.com/origindiamond/n/nde71832519de)
- [カプコン9697トレードレポート2025年9月](https://note.com/origindiamond/n/nb2d40fe03a53)

## 正規化ルール

### 月足前提

- `上昇初動候補`
  - 60V、長期線上抜け、横ばい後の上放れ狙い
- `天井圏反落候補`
  - 天井ボックス圏、2ヶ月下落後の反発狙い、前高値圏での失速狙い
- `横ばい継続`
  - 上昇/下落どちらにも決め切れず、ボックス長期化を想定
- `想定外停滞`
  - 想定月数・本数を超えてもトレンドが出ず、時間切れ撤退を検討すべき局面

### 日足シグナル

- `コマ`
- `窓下げ`
- `窓埋め`
- `連続陰線`
- `連続陽線`
- `20日線接触/割れ/回復`
- `前高今安`
- `全消し陰線/陽線`

### 玉操作

- `買い追加`
- `買い利確`
- `買い損切り`
- `売り追加`
- `売り利確`
- `売り損切り`
- `両建て維持`
- `全撤退`

記録時の原則:

- 1レコード = 1局面
- `直前ポジション` は必ず `売x:買y` で記録する
- `実行操作` は必ず数量差分で記録する
- `再現ルール候補` は記事の原文ではなく条件文に直す

## 仮説ルール集

### 初期エントリー条件

- 月足が `上昇初動候補` で、日足が `コマ` や `20日線回復待ち` の場合は、小さめの両建てで開始する
- 月足が `天井圏反落候補` で、日足が `20下20消化後のコマ` なら、買い優位の反発狙いで入る
- 月足が `天井圏反落候補` で、方向感がまだ曖昧なら、売り優位の小口両建てで始める

### 追加条件

- `上昇初動候補` で `横並びコマ` や `戻し陽線` が出たら、売りヘッジを残しつつ買いを追加する
- `天井圏反落候補` で `連続陰線` が出たら、売り優位を増やして下落本線に寄せる
- すでに十分仕込めている状態で `上抜け陽線` が出た場合は、無理に追加せず維持を優先する
- `前安値割れ後の戻し陽線` は、下落トレンドへの売り積み直し候補として扱う

### 利確条件

- 継続上昇の想定を崩す `大陰線` や `7日割れ陰線` が出たら、主力買いを大きく利確して再エントリー余地だけ残す
- 反発狙いの後に `コマ全消し` が出て伸びが止まったら、短期終了として全利確を優先する
- `窓下げコマ` で長期化の匂いが強まったら、買いを大きく落として資金拘束を減らす

### ヘッジ切替条件

- `上昇初動候補` でも `大陰線` が出たら、買いを削って売り比率を半ヘッジ付近まで引き上げる
- 強い `大陽線` が出て上昇再開の可能性が高まったら、含み益の売りを利確して買い優位へ戻す
- `天井圏反落候補` で `窓下げ` や `コマ全消し陰線` が出たら、逆張り買いを利確して売り側へ寄せる
- `下から始まるコマ` が出ても強い陽線に発展しない場合は、買いを損切りして売りを戻す

### 時間切れ撤退条件

- 想定した上昇月数を超えてもトレンドが出ない場合は、日足改善を待たず一度全撤退する
- `想定外停滞` の局面では、下髭や単発陽線だけで追加せず、翌日の確認を待つ

## 一次採用ルール

CSV上で3回以上出ているため、現時点で一次採用とするルール:

- `コマ` を起点に小さく両建て開始する
- `窓下げ` や `大陰線` で買いを落として売りを厚くする
- `強い陽線` で売りの利確を進めて買いへ戻す
- `コマ全消し` で伸びが止まったら利確を優先する

## 観察メモ

- `7日割れ陰線` は大平洋金属では強い利確シグナルだったが、現時点では単発
- `20下20消化` は宝HDで有効に見えるが、他記事で反復確認がまだ足りない
- `前高今安` はカプコンで反発根拠に使われているが、売り転換と衝突しやすく単独では弱い
- `価格が安いので500株=1` のような単位調整は、手法そのものではなく資金管理ルールとして別管理した方がよい

## 失敗類型

### 想定外停滞

- システナでは `2026-02-04` に全撤退しており、記事内でも `4ヶ月目で上昇しなかったので一回撤退した方が良かった` と振り返っている
- この失敗は、日足の一時改善より `月足シナリオの時間切れ` を優先すべき例として扱う

### 資金配分過多

- 大平洋金属では、月足の強さを信じて買いを積み過ぎた直後に大陰線対応が必要になっている
- 上昇初動候補は魅力が強いぶん、長期化前提の資金拘束リスクを別枠で管理する必要がある

### 利確後の再乗り遅れ

- 宝HDや大平洋金属では、利確後に横ばい再仕込みを待つ構えが何度も出る
- 利確自体は正しくても、再エントリー条件が曖昧だと検証不能になる

## 比較メモ雛形

自分の過去トレードと照合する時は、最低10件を同じ列で並べる。

### 一致している部分

- 月足前提を置いてから日足で玉を調整しているか
- 片張りではなく両建て比率で管理しているか
- 利確を `結果` ではなく `局面変化への対処` として使っているか

### 差分として見る部分

- 同じ `窓下げ` でも自分は買い増し寄りか、ヘッジ増し寄りか
- `コマ全消し` を利確シグナルとして使っているか
- `時間切れ撤退` を日柄で切れているか
- 想定と逆行した時に、数量調整で済ませるか、全撤退するか

### 取り入れる候補

- 局面ごとに `直前ポジション` と `数量差分` を必ず残す
- `月足シナリオの時間切れ` を独立した撤退ルールにする
- 利確後の再エントリー条件を `コマ/窓下げ/戻し陽線` のどれで行うか固定する

### 見送る候補

- 単発の言い回しだけで一般化したルール
- 記事中で後悔として書かれている追撃買い
- 決算跨ぎのように、イベント要因が強く混ざる局面をそのまま標準ルール化すること

## 次の使い方

1. CSVに自分の過去トレードを10件以上追記する
2. `月足前提_正規化` と `日足シグナル` が一致する局面だけを並べる
3. `実行操作` の差分を見て、どの操作が期待値改善につながるかを別集計する

前提:

- この研究では銘柄発掘の再現は扱わない
- note記事の表現は完全定量化せず、半裁量ルールとして扱う
- 期待値の厳密検証は次段階とし、まずは `同じ局面で同じ操作に寄るか` を見る
