## 発見された問題

### 1. 保有銘柄リストが表示されない (「保有銘柄はありません」)

**原因:**
- 取引履歴データがインポートされていない
- `trade_events` テーブルにデータが存在しない
- `positions_live` テーブルが空

**状態:**
- データベーステーブルは正常に作成済み
- 取引履歴CSVのインポート待ち

**解決方法:**
1. フロントエンドの「保有/履歴」画面を開く
2. 楽天証券またはSBI証券の取引履歴CSVファイルをアップロード
3. システムが自動的に以下を実行:
   - 取引イベントをパース
   - `trade_events` テーブルに保存
   - `rebuild_positions()` を実行して保有銘柄を計算
   - `positions_live` テーブルを更新

### 2. 決算発表日・権利落ちデータの取得失敗 (「更新失敗」)

**原因:**
- 必要なPythonライブラリが不足していた:
  - `openpyxl` - Excel(.xlsx)ファイルの読み込みに必要
  - `jpholiday` - 営業日判定に必要

**修正内容:**
✅ **修正完了** - 不足していたライブラリをインストールしました
```bash
pip install openpyxl jpholiday
```

**次のステップ:**
1. フロントエンドから「イベント更新」ボタンをクリック
2. または、以下のAPIを呼び出す:
   ```
   POST http://localhost:8000/api/events/refresh
   ```
3. システムが以下を実行:
   - JPXのウェブサイトから決算予定・権利落ちのExcelファイルをダウンロード
   - データをパースして `earnings_planned` と `ex_rights` テーブルに保存
   - 各銘柄の詳細画面にイベント情報が表示されるようになる

## トラブルシューティング

### イベント更新が失敗する場合

**確認事項:**
1. インターネット接続を確認
2. ファイアウォールがHTTPS通信を許可しているか確認
3. JPXのウェブサイト (jpx.co.jp) にアクセスできるか確認

**エラーメッセージ別の対処法:**

- `earnings_excel_urls_not_found` または `rights_excel_urls_not_found`
  → JPXのウェブサイトからExcelファイルのURLを自動検出できませんでした
  → 環境変数で手動でURLを指定することも可能

- `URLError` または `timeout`
  → ネットワーク接続の問題
  → プロキシ設定が必要な場合は環境変数で設定

- `ModuleNotFoundError` または `pandas_not_installed`
  → 必要なライブラリが不足
  → `pip install pandas openpyxl jpholiday` を実行

### 保有銘柄が正しく表示されない場合

**確認事項:**
1. 取引履歴CSVが正しくインポートされたか確認
2. CSVのフォーマットが対応しているか確認
3. 診断スクリプトを実行して状態を確認:
   ```bash
   python diagnose_issues.py
   ```

**よくある問題:**
- 数量がマイナスになる → `has_issue` フラグが立ち、「要確認」と表示されます
- 銘柄名が表示されない → `tickers` テーブルに銘柄情報がない可能性
- 日付が「不明」 → 取引日時の解析に失敗した可能性

## 技術的な詳細

### データフロー

#### 取引履歴インポート
```
CSVアップロード
  ↓
parse_rakuten_csv() または parse_sbi_csv()
  ↓
trade_events テーブルに保存
  ↓
rebuild_positions()
  ↓
positions_live テーブル更新 (現在の保有)
position_rounds テーブル更新 (過去の取引)
```

#### イベントデータ更新
```
イベント更新リクエスト
  ↓
fetch_earnings_snapshot()
  ├─ JPXから決算予定Excelをダウンロード
  ├─ Excelをパース
  └─ earnings_planned テーブルに保存
  ↓
fetch_rights_snapshot()
  ├─ JPXから権利落ちExcelをダウンロード
  ├─ Excelをパース
  └─ ex_rights テーブルに保存
  ↓
events_meta テーブル更新 (最終更新日時)
```
