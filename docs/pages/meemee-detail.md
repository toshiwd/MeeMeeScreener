# MeeMee Detail Page

## 目的

一覧で選んだ 1 銘柄について、環境認識、傾向認識、エントリータイミング、建玉状況、補足メモを 1 画面で確認する主画面である。

- TradeX の研究作業場ではない
- 重い学習、検証、探索は置かない
- 役割は可視化と確認である

## 画面構成

基本の時間軸は上から次の 3 層とする。

- 月足: 環境認識
- 週足: 傾向認識
- 日足: 実行タイミング確認

3 つは独立した別チャートではなく、同じ銘柄の時間軸として連動する。

## チャート表示

- ローソク足を基本とする
- 移動平均線を重ねる
- 必要な補助表示を重ねる
- ボックス / レンジ枠を表示する
- イベント情報を重ねる
- 必要に応じてオーバーレイを表示する

優先度:

- 日足が最も情報量が多い主チャート
- 週足と月足は上位環境確認の主役

## 分析モード

TradeX で研究・publish されたロジックの判定理由を可視化するモードである。

- 買い / 売り判定の根拠
- どこで判定が出たか
- 目視確認できること

AI コメントは将来追加してもよいが、説明レイヤーであり本体ではない。

## 分析マーカー

- 売買判定マーカーは日足のみ
- 形状はシンプルな `◯`
- 赤 = 買い
- 緑 = 売り
- 週足と月足には同じ売買実行マーカーを出さない

## 週足・月足の補助表示

補助表示の候補は次のとおり。

- レンジ枠
- ボックス
- 長期支持抵抗帯
- 高次足で意味の強いシグナル

ただし、最初から増やしすぎない。画面の主役は時間軸の確認である。

## オーバーレイ

カーソル追随で対象日の詳細を確認する。

例:

- 判定理由
- 価格情報
- シグナル
- 建玉状況
- メモ
- イベント情報

## 財務モード

EDINET、TDNET、貸借関連などの無料データ閲覧モードである。

- 閲覧中心
- 重い解析や学習は持たない
- 画面遷移の主目的は確認である

## 建玉 / ポジション表示

- 狭いスペースでも見やすいよう、分数のような表現を使う
- 楽天 / SBI の取引履歴に対応する
- 現在建玉だけでなく日次時点状態も確認できるようにする

## メモ

- 日足の特定日にメモを残せる
- AI 画像分析の補足にも使える
- 売買理由、違和感、当時の判断材料を残す
- 分析ロジック本体とは分離する

## 類似チャート

月足、週足、日足のローソク足形状、移動平均線形状、傾き、位置関係が似ている過去局面を探す。

- 目的は未来のシナリオ候補の観測
- 予言ではない
- MeeMee では閲覧・比較用
- TradeX では研究、検証、特徴量改善にも活用可能
- 左 = 現在の基銘柄
- 右 = 過去の類似銘柄
- 類似銘柄は類似起点日を右端基準で表示する
- 未来側チャートもドラッグ確認可能
- 次の比較銘柄も保持できる前提とする

## confirmed / provisional 境界

- `confirmed` = 解析、判定、根拠に使ってよい
- `provisional` = 場中の補助表示のみ
- 暫定値表示時は UI 上で明示する
- 分析判定やランキング根拠には使わない

## 銘柄移動時の時点保持

- 過去日時点を見ているとき、他銘柄へ移動しても可能な限り同じ日付帯を維持する
- 完全一致がなければ最も近い有効日へ寄せる
- 明示的に最新へ戻したときだけ最新へ戻す
- 一覧から通常遷移時は最新優先でもよい

## 置かないもの

- TradeX の重い研究処理
- 学習ジョブ
- walk-forward 詳細管理
- 大規模比較検証 UI
- 長文研究ログ
- ランキング生成そのもの

## 定義

「月足は環境、週足は傾向、日足は実行。個別詳細ページはその3層を同時に確認するための画面。」

## Trades からの遷移時の時点保持

Trades ページから遷移した場合は、最新優先ではなく、受け取った対象日付帯を優先する。
完全一致がなければ最も近い有効日へ寄せる。

- まず日足を中心に開く
- 週足・月足は補助的に同期する
- 明示的に最新へ戻したときだけ最新へ戻る
- 一覧から通常遷移した場合の最新優先ルールとは分けて扱う

## 初版遷移ルール

Trades ページからの遷移時は、受け取った `anchorDate` を優先する。

- まず日足を中心に開く
- 週足・月足は補助的に同期する
- 完全一致がなければ最も近い有効日へ寄せる
- 明示的に最新へ戻したときだけ最新へ戻る
- 一覧から通常遷移した場合の最新優先ルールとは分けて扱う
