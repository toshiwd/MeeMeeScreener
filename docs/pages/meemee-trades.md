# MeeMee Trades Page

## 目的

このページの主役は建玉確認である。
取引履歴は、その建玉状態の根拠を追うための表として置く。

取引履歴と建玉状況を確認するためのページである。

この画面は、次を確認する役割に絞る。

- 何をいつ売買したか
- その結果、今どんな建玉になっているか
- 過去日時点ではどんな建玉だったか

一覧ページの役割は「見る銘柄を選ぶこと」。
個別詳細ページの役割は「チャートと判定理由を確認すること」。
Trades ページは、その間にある取引履歴・建玉確認の専用画面とする。

TradeX の研究画面ではない。
重い学習、検証、walk-forward、ランキング生成は置かない。

## 画面の基本構成

主役は次の 3 つとする。

- 取引履歴一覧
- 建玉サマリー
- 日次建玉推移

必要に応じて、銘柄別表示と全体表示を切り替えられるようにする。

## 取引履歴一覧

取引履歴一覧は、楽天証券 / SBI証券から取り込んだ履歴を確認するための表である。

表示対象の基本項目は次のとおり。

- 約定日
- 受渡日
- 証券会社
- 銘柄コード
- 銘柄名
- 売買区分
- 現物 / 信用区分
- 数量
- 単価
- 手数料
- 約定金額
- メモ / 補足情報

一覧は「履歴の事実」を見る場所とする。
ここでは分析判定やランキング理由は混ぜない。

## 建玉サマリー

建玉サマリーは、現在時点または指定日時点の建玉を確認する領域である。

表示原則は次のとおり。

- MeeMee 内部の建玉表記は `売-買` で統一する
- 狭い領域でも読みやすいよう、比率表示を優先する
- 必要に応じて実数量も別表示する
- 買い平均単価と売り平均単価は分けて扱う
- 現物 / 信用は分離表示する

例:

- `2-6` = 売り 2、買い 6
- 株数表示は別で `200 / 600` のように表示してよい

## 日次建玉推移

このページでは、現在建玉だけでなく、過去日時点の建玉状態も確認できるようにする。

用途は次のとおり。

- ある日の判断時に、実際どのくらいポジションを持っていたか確認する
- 建玉がどのタイミングで増減したかを追う
- 売買履歴の正規化結果が正しく建玉へ反映されているか確認する

ここで使う建玉は、取引履歴そのものではなく、取引履歴から復元した日次建玉の派生データとして扱う。

## 使う契約

このページが直接使う契約は次の 2 つである。

- `trade_history_normalized`
- `position_snapshot_daily`

役割分担は明確に分ける。

### trade_history_normalized

- 楽天 / SBI の CSV を正規化した履歴本体
- 取引の事実を保持する
- 売買、日付、数量、単価、手数料などを正規化する

### position_snapshot_daily

- 取引履歴から復元した日次建玉
- ある日時点の売り / 買い残高、平均単価などを保持する
- 履歴そのものではなく派生契約である

この 2 つを 1 テーブルで兼ねない。

## フィルターと検索

最低限ほしい絞り込みは次のとおり。

- 銘柄コード / 銘柄名検索
- 証券会社
- 現物 / 信用
- 売り / 買い
- 期間指定
- 建玉ありのみ

フィルターは一覧ページと同じく、その場限りでもよい。
恒久保存は必須ではない。

## 並び順

基本の並び順は次のとおり。

- 約定日 降順
- 受渡日 降順
- 銘柄コード順
- 建玉量順

デフォルトは約定日降順でよい。

## 銘柄別表示

このページは、全履歴を俯瞰する使い方と、1 銘柄に絞る使い方の両方を想定する。

銘柄別表示では次を優先する。

- その銘柄の全履歴
- 現在建玉
- 日次建玉推移
- 平均単価推移
- 補足メモ

個別詳細ページへ遷移できる導線を持ってよい。
ただし、このページ自身はチャート主役にしない。

## CSV 取り込みとの関係

CSV 取り込み自体の仕様は別文書に分ける。

このページの役割は、インポート結果を確認し、建玉へどう反映されたかを見ることにある。

そのため、必要なら次の状態表示を持ってよい。

- 最終取込日時
- 取込元証券会社
- 取込件数
- エラー件数
- 再計算日時

ただし、詳細な import 実装ルールは `trade-history-import.md` 側で管理する。

## 置かないもの

このページに置かないものは次のとおり。

- TradeX の重い研究処理
- ランキング生成
- 学習ジョブ
- walk-forward の詳細
- 類似チャート比較
- 長文の研究ログ
- AI の長い解説文

このページは、取引履歴と建玉の確認画面であって、研究画面でもチャート分析画面でもない。

## 詳細ページへの遷移

過去の取引履歴行をクリックしたときは、同一銘柄の個別詳細ページへ遷移する。
遷移時は、選択した取引の約定日を基準にチャート期間を合わせて開く。

- 必要に応じて受渡日ではなく約定日を優先してよい
- 日足を主表示にする
- 週足・月足はその文脈に同期する
- 取引履歴一覧は建玉確認の根拠を追うための表であり、クリック遷移はその確認導線である

## 初版遷移ルール

初版では、過去の取引行をクリックしたときの詳細ページ遷移は単純に扱う。

- `code + anchorDate` のみを detail に渡す
- `anchorDate` はクリックした行の約定日とする
- 受渡日例外は設けない
- detail はその日付を中心に日足を開く
- 週足・月足は補助同期とする
