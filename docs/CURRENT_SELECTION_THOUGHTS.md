# 銘柄選定の思想（暫定・2026-02-07時点）

この文書は**確定ルールではない**。  
現時点でバックテストを回して、比較的再現性が見えた「途中経過の答え」を整理したメモ。

## 1. いまの仮説（選定思想）
- 目的は「毎月1〜2銘柄でも、期待値が高い局面を優先して入る」こと。
- 全パターンを均等に使うより、**勝ちやすいセットアップを絞る**方が有効。
- 買いと売りは同一ロジックで混ぜるのではなく、**有効だった型だけを残す**。
- 直近相場に引っ張られないため、3年だけでなく10年で確認し、年ごとの偏りを見る。

## 2. 現在の有力セットアップ（暫定）
- 買い: `long_pullback_p3`
- 売り: `short_downtrend_p4`
- 月次上限: `max_new_entries_per_month=2`
- 日次上限: `max_new_entries_per_day=2`

## 3. レジーム判定（今回追加）
- `above60` 比率（当日ユニバースで終値が60MA上にある比率）を使って市場地合いを判定。
- 平滑化: `regime_breadth_lookback_days=40`
- 買い許可: `breadth >= 0.55`
- 売り許可: `breadth <= 0.45`
- これにより、地合いと逆方向のエントリーを抑制する。

## 8. 2026-02-07 追加（ランキング精度チューニング）
- `hybrid` ランキングで、以下を満たす候補を優先:
  - 方向整合した `EV20_net`（上昇: `>= +0.3%` / 下落: `<= -0.3%`）
  - 方向整合した確率（既存 `min_prob_up/down` 以上）
  - トレンド整合（上昇: `close > ma20 > ma60`、下落: `close < ma20 < ma60`）
  - `|close - ma20| / ma20 <= 12%`
- 不足時の補充も「EV符号」と「トレンド整合」を満たす候補のみを使用。
- これにより、弱い逆行候補の混入を抑え、候補数より精度を優先する仕様に寄せた。
