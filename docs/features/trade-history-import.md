# Trade History Import

## 目的

楽天 / SBI の取引履歴 CSV を MeeMee の内部表現へ安全に取り込む契約を固定する。

## 取り込み先

- 正規化先は `trade_history_normalized`
- 元 CSV は別に保持する
- 画面表示用の集計や建玉復元は別責務にする

## 正規化ルール

- 現物 / 信用を分けて保持する
- 買い / 売りを分けて保持する
- 約定日、受渡日、手数料、数量、単価を正規化する
- MeeMee 内部の建玉表記は `売-買` に統一する

## 分離するもの

- `trade_history_normalized` は履歴の正規化テーブル
- `position_snapshot_daily` は履歴から作る日次派生契約
- 1 テーブルで両方を兼ねない

## 既存ルールとの整合

- 既存コードの暗黙ルールを壊さない
- broker 固有の列名や並び順に依存しない
- 不明列は直ちに捨てず、必要なら raw 側へ残す

## エラー扱い

- CSV 形式が壊れている場合は明示的にエラーにする
- 不明な銘柄コードや日付は黙って補正しない
- 取込失敗を成功に見せない
