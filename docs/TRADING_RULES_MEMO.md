# トレードルール追記メモ（ユーザー入力保存）

更新日: 2026-02-07

補足（思想メモ）: `docs/CURRENT_SELECTION_THOUGHTS.md`

## 1. 資金枠
- 銘柄ごとの投入上限: `5,000,000円`
- ロング＋ショートの両建て含む想定
- 上限超過になる増し玉は不許可（最終約定をスキップ）

## 2. ポジション単位（2-3-5）
### 基本ユニット（株数）
- 初回IN: `+200株`
- 追加1: `+300株`（合計 `+500株`）
- 追加2: `+500株`（合計 `+1,000株`）

### 倍量ユニット（低位株用）
- 初回IN: `+400株`
- 追加1: `+600株`（合計 `+1,000株`）
- 追加2: `+1,000株`（合計 `+2,000株`）

### バックテスト分岐
- 倍量ユニット利用有無はフラグで分岐

## 3. ヘッジ（最低1/5）
- 常に反対玉の株数（または金額）が主玉の `1/5` 以上
- 例: ロング合計 `1,000株` ならショート `200株` 以上
- ヘッジ込みで資金枠 `5,000,000円` に収める

### ヘッジ評価モード
- `A`: 同一銘柄ヘッジ（同一OHLCで評価）
- `B`: 指数ヘッジ（ヘッジ銘柄OHLCが必要）

## 4. 執行タイミング
- 仕掛け、増し、利確、損切りはすべて終値確定（引け成）
- 場中の揺れは無視

## 5. シグナル（テクニカルのみ）
- ローソク足、移動平均、出来高（20日平均）
- 飯塚式5パターン、決定打（B/G/M 2/3）をエントリー／増し／撤退条件に使用
- 本数カウント（7下/20下/60下）は終値ベースで更新し、局面フィルタに使用

## 6. 増し玉許可条件（機械条件）
- 初回 `+200株`: エントリーパターン成立でIN
- 追加 `+300株`: トレンド方向更新（売り: 前日安値更新、買い: 直近高値更新）または決定打成立
- 追加 `+500株`: 戻り失敗（短期線リテストで反転）または主要ライン（20MA/直近安値）割れ・維持確定

### 増し前チェック（必須）
- `a`: ヘッジ `>= 1/5` を維持できるか
- `b`: 資金枠 `5,000,000円` を超えないか
- `c`: イベント回避日（決算・権利日・重要イベント `T-2〜T+1`）に該当しないか

## 7. 利確と損切り（玉操作）
- 利確: まず半分利確 → 伸びないなら全利確（2段で落とす）
- 損切り: シナリオ否定（終値で重要ライン回復/割れ）で全撤退
- 部分利確は株数変換
- 例: ロング合計 `1,000株` なら `500株` 利確 → 残り `500株`

## 8. バックテスト実装メモ（数式）
### 建玉評価額（概算）
- ロング額: `Σ(保有株数 × 終値)`
- ショート額: `Σ(保有株数 × 終値)`
- グロス: `ロング額 + ショート額 <= 5,000,000円`

### ヘッジ比率
- `min(ショート株数, ロング株数) / max(ショート株数, ロング株数) >= 0.2`
- 指数ヘッジモードでは株数比ではなく金額比率に切替
